Eclipse RSS Reader

The Eclipse RSS Reader is a set of Eclipse plug-ins that allow you to subscribe to and browse on-line RSS (aka Rich Site Summary, aka Really Simple Sindication, aka RDF Site Summary) feeds. The New Channel Wizard helps you create new Channels in your workspace using your RSS feed as a source. You can browse your workspace Channel hierarchy using the Channel Navigator, or view a particular Channel's items in the Channel Detail view. You can modify each Channel's properties using the Properties dialog, and provide default Channel settings and other configuration values in the Preferences. Finally, you can search your Channels using the Search dialog.

Link browsers

You may choose, using the Channel Properties dialog, how to view the contents of channel and item links. By default, two methods are available:

  1. Using the configured Help Browser
  2. Using a file editor that understands the article format (e.g., HTML)

In addition, users of WSAD running under MS-Windows have an option to use its embedded web browser. 3rd-party providers may offer additional link browsers.

RSS feed translators

RSS feeds based on various formats and versions are translated into a uniform channel model, which allows the various RSS views to present their contents in a uniform fashion. The default translator understands RSS versions 0.90, 0.91, 0.92, 1.0, and 2.0. 3rd-party providers may offer translators for new RSS versions, or entirely different formats.

Periodic updates

For each channel you may specify an interval, in minutes, which dictates how often the channel's contents are updated from its source feed. After an update, any new items are flagged and displayed with different icons and colors in the various RSS views, until they are read (i.e., opened using a link browser).

Channel content caching

RSS channels are represented in the workspace as files, which contain information on where to get the updates, how often, etc. The channel content, after an update, is cached in a temporary location outside of workspace. Thus it is possible to share the channel files using a Version Control System (e.g., CVS); each user then gets its own copy of the channel contents.
