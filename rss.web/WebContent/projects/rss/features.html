Features

At the moment, the Eclipse RSS Reader provides the following functionality:

1. Uniform channel model

RSS Feeds of various versions are coerced into a uniform channel model, which is persisted in the workspace. Third-party plug-ins may contribute "translators" for various channel formats. The default implementation handles RSS 0.90, 0.91, 0.92, 1.0, and 2.0. In case more than one registered translator understands a format, the user is given a choice to select one they wish to use. In the future, annonymous elements external to the model will be exposed via untyped property interface (map), thus making them available to third-party plug-ins that may understand and use them in a specific way.

2. Periodic updates

Based on user specification, channels are periodically updated from the source. All viewers are refreshed to display up-to-date information. In the future, markers will be used to identify changes and allow viewers to highlight them.

3. Property page

Individual channel files have a property page with RSS-specific options, such as update interval, source translator, and link browser.

4. Preference page

Preference page allows the user to specify default values for update interval and link browser. In addition, default link browsers (help and editor) have their own category; the former allows the user to select which configured help browser to use, and the latter which editor to use to browse articles.

5. New Channel wizard

Allows users to create new channels from an external RSS feed.

6. Channel Navigator

Allows users to navigate channels in a project/folder hierarchy.

7. Channel Detail view

Allows users to view details of selected channel, such as title and description. Supports sorting by column.

8. Pluggable Link Browsers

Users may choose how to view the contents of news articles. By default, two methods are available: 1). Using the configured Help Browser, or 2). Using a file editor that understands the article format (e.g., HTML). In addition, users of WSAD running under MS-Windows have an option to use its Web Browser Editor. This option is specifiable on a per-channel basis. 3rd-party provides may offer additional methods via the provided extension point.

Future Plans

The following items are currently on my to-do list:

  • Highlight items that were modified after an update until they are read.
  • Filter HTML tags from title and description fields.
  • Expose data external to the core model via mapped properties so other plug-ins may consume them.
  • Allow user to select channel image.
  • Provide migration path for users of other RSS Readers (e.g., listen on port 8888 and process AmphetaDesk-compatible URL requests to subscribe to channels).
  • Write user documentation, context-sensitive help, JavaDoc... :-)
  • Internationalization (externalize strings into resource bundles).
