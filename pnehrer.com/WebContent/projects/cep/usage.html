Using the Plug-in

At the moment the primary functional feature of the plug-in is generation of Java source code from XML Schema. The plug-in serves as a front-end to Castor's Source Generator. After right-clicking on an *.xsd file the user will find two Castor-related menu items in the context menu: 1). Generate Castor XML Sources..., and 2). Run Castor Automatically.

As its name implies, the first menu item allows for generation of Castor XML sources based on the selected XML Schema file. When the menu item is selected, a dialog box will ask the user to provide values for the Source Generator as well as where to save the generated files. After clicking OK the generator is run. If the output directory is within the project, the workspace is refreshed and generated files appear in the navigator view.

The second menu option turns on the plug-in's incremental builder for the selected XML Schema file (or files). This means that whenever the project is built, or whenever the *.xsd file changes, the incremental builder will run Castor Source Generator and produce files according to the contents of the XML Schema file. The builder keeps track of changes and adds or deletes generated files accordingly. The generator can be configured using the file's property page; when the XML Schema file's property dialog is open, a Castor tab will appear, allowing the user to configure output folder as well as other generator configuration options.

Default configuration options can be specified in the Castor tab of the workspace Preferences dialog.

Please note that the generated files will not compile unless Castor libraries are added to the project's build path. You may use your own Castor jars or those provided with the plug-in in its lib directory.
