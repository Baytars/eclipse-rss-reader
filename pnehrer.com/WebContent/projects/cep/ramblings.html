What I still don't get

I learned a great deal about Eclipse while working on this small project. However, here some of the things I still don't get:

  • The pattern of various element "ids", such as object contributions, pop-up menu actions, builders, natures, etc. It seems that some follow the namespace/local name paradigm, where the plug-in id is the namespace and the element id (e.g., the builder id) is the local name. Thus to refer to a builder you must provide the whole name. However, some other elements, such as pop-up menu action, don't seem to follow this (judging from the wizard-generated ids).
  • When and how the resource changes are generated. Why do I get two CHANGED deltas in the builder when I "touch" a file? Does changing a file's persistent property fire an event?
  • Why are some paths workspace-relative, some project-relative, and some local-filesystem-relative? Why is a "root path" workbench-relative?
  • When should I use the IAdaptable interface on resources? Should always use instanceof first, then use IAdaptable? Why not always IAdaptable?

I'm sure there are more. Maybe I'll add them when I remember. If anyone can provide any clarification, please feel free.
