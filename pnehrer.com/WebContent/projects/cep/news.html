News

2003-07-25

Updated to version 1.2.1. This release fixes a bug -- incremental builder outputs to a location under the workspace root rather than the actual project location, which may not be under the workspace tree.

Plugin's compatibility with WSAD 5.0.1 (and thus IBM's JDK 1.3) running on Windows 2000 has also been verified.

2003-07-24

Released version 1.2.0. This release sports the addition of a context menu action that allows the user to generate Castor XML Sources from the selected file without having to do a project build. The file does not have to have Castor XML Source Generation enabled and configured; all required settings are obtained via a dialog box, which pops up when the menu action is selected. The feature also allows one to generate the files into a location that is outside of the project directory structure.

This release has been developed and tested on Linux 2.4.21 with Eclipse 2.1.1 and Sun JDK 1.4.2. It may not be backward-compatible with previous versions of Eclipse or Java.

2003-07-12

Version 1.1.0 is released. This release includes the following changes:

  • added preference page to control default generator settings
  • added new generator settings to Castor schema file property page to override default preferences
  • created Eclipse-friendly source generator dialog class to interact with the user whenever the generator needs user input or report an error/warning
  • added logging of previously ignored exceptions
  • updated Castor's SourceGenerator to accept dialog implementation other than the default ConsoleDialog
  • upgraded project structure for Eclipse 2.1.1
2003-04-01

Plug-in version 1.0.1 is released. This maintenance release contains refactoring that makes this plug-in compatible with JDK 1.3.1 and Eclipse 2.0 (e.g., WebSphere Application Developer 5).

2003-03-24

Castor Plug-in for Eclipse v1.0.0 is released. This initial release provides the basic functionality that makes this plug-in useful. XML Schema files can be marked as supporting Castor generation. Whenever changes to such files occur, the generation process is kicked off. Generated Java files are automatically tracked by the workspace.
